User model changes (apply to src/domains/users/models.py) — supersedes
the previous USER_MODEL_DIFF.md (Crossmint-Auth version).

1. DELETE the password column (same as before — still no local passwords):

    password = Column(String, nullable=True)

2. DELETE crossmint_user_id. There is no Crossmint-issued identity
   anymore — Crossmint never authenticates anyone in BYOA, it only
   provisions a wallet against OUR already-authenticated user, addressed
   by OUR user id embedded in our own JWT. Nothing needs to look a user
   up BY a Crossmint id; flow_address (the resulting wallet) is the only
   Crossmint-derived field we still keep.

    crossmint_user_id = Column(String, unique=True, index=True, nullable=True)

3. ADD firebase_uid — the new external-identity column, replacing
   crossmint_user_id's old role, but for Firebase/Google instead:

    firebase_uid = Column(String, unique=True, index=True, nullable=True)

4. CHANGE email to nullable. Phone and email are now equally-primary
   per the product decision — a user who registers via phone OTP only
   may never have an email on file at all.

    Before:
        email = Column(String, unique=True, index=True, nullable=False)
    After:
        email = Column(String, unique=True, index=True, nullable=True)

   phone_number stays as it already is (unique, nullable=True) — already
   correctly shaped for this, no change needed there. Note the DB-level
   invariant this implies but does NOT enforce on its own: every user
   must have at least one of (email, phone_number, firebase_uid) set.
   Enforce this at the application layer (AuthService never constructs a
   User without one) and optionally add a CHECK constraint in the
   migration for defense in depth:

    CHECK (email IS NOT NULL OR phone_number IS NOT NULL OR firebase_uid IS NOT NULL)

5. KEEP unchanged: flow_address, wallet_provider, is_email_verified,
   is_phone_verified, is_video_verified, wallet_activated, is_verified,
   and all onboarding/preference fields.

6. username stays unique/required, auto-generated by AuthService on
   first provisioning regardless of which channel was used.
